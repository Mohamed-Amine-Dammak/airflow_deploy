Security & Access Control Update

Airflow Orchestration Builder - v0.2

Authentication, role-based access control, custom role management, and admin user governance are now live.

Open Builder View v0.1 Notes

Highlights in v0.2

  • Introduced full login-based authentication for all app and API routes.
  • Added predefined roles and permission-driven feature gates across UI and backend endpoints.
  • Added admin screens and APIs to create users and assign one or more roles per user.
  • Added custom role creation with editable privilege and permission sets.
  • Expanded Airflow Connections capabilities with create, edit, and delete operations.
  • Upgraded top navigation and action grouping for faster access to operational pages.

Authentication Implementation

  • Login now validates credentials against stored password hashes using Werkzeug secure hash verification.
  • Successful login establishes a Flask session with authenticated state, username, and resolved role list.
  • A global before_request guard enforces authentication for all protected pages and API routes.
  • Unauthorized API requests return 401; unauthorized pages are redirected to the login screen.
  • Logout clears session state to immediately revoke access in the current browser session.

Predefined Roles

  • admin: full access across builder, runs, retries, logs, connections, users, roles, and admin dashboard.
  • developer: pipeline design/edit/save/validate, auto-layout, DAG generation, and connection viewing.
  • operator: DAG generate/run/retry operations, task retry, logs viewing, and connection viewing.
  • viewer: read-only builder visibility and runtime/log visibility without edit privileges.
  • connection_manager: connection listing plus create/edit/delete management workflows.

Permission Model

  • Actions are protected with permission checks server-side via dedicated guard helpers.
  • UI buttons and menus are dynamically enabled/disabled using current session permissions.
  • Permission keys include builder, pipeline, DAG run/retry, logs, connections, users, roles, and admin domains.
  • Role permissions are aggregated per user, enabling multi-role access composition.

Custom Roles

  • Admins can create custom roles from the role management page or API endpoints.
  • Each custom role supports a description, human-readable privileges, and explicit permission keys.
  • If permissions are omitted, the app infers best-effort permissions from provided privileges.
  • Custom roles can be edited, renamed, and deleted; user assignments are remapped safely on role rename.

Admin Privileges & User Management

  • Admin dashboard centralizes platform metrics, user inventory, role catalog, and recent run insights.
  • Admins can create users with secure password hashing and assign one or many roles at creation time.
  • Admins can edit existing users, update role assignments, and delete non-system users.
  • System-level admin identity remains protected from accidental deletion.

Additional UX Changes

  • Top navigation was upgraded for clearer action grouping and better day-to-day operability.
  • Actions menu now includes a dedicated Releases grouping for historical notes.
  • Status and operational shortcuts remain aligned with permission-based visibility.

Airflow Connections Expansion

  • Added create, edit, and delete workflows for supported Airflow connection types.
  • Connection management respects role permissions (connections_view and connections_manage).
  • Improved error handling around API auth failures and connection mutation responses.

Compatibility Notes

  • v0 and v0.1 builder/run capabilities remain available with role-based gating now applied.
  • Predefined and custom roles can coexist, allowing incremental migration from simple to granular access.
  • Release notes remain accessible for all three versions: v0, v0.1, and v0.2.
