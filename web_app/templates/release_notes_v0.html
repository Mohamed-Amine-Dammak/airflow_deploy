First Release

Airflow Orchestration Builder - v0

Initial production-ready baseline for visually building and generating sequential Airflow DAGs.

Open Builder Back Home

What v0 Includes

  • Interactive drag-and-drop canvas with visual modules and live connection arrows.
  • Sequential pipeline modeling with validation (single chain, one start, one end, no cycles).
  • Dynamic right-side configuration forms for each module type.
  • Pipeline settings with scheduling mode selector (Manual, Preset, Cron, Interval, Event/Dataset, Custom).
  • Template-based DAG rendering with Jinja2 and readable generated Python output.
  • One-click DAG generation, preview, and copy to Airflow DAG folder.
  • Save/load pipeline JSON with workspace status feedback.

Supported Modules

n8n Talend Dataform Cloud Run Azure Upload Azure Download Azure Blob Sensor Power BI SFTP Upload SFTP Sensor Y2 Scheduled Task

UX Improvements Included

  • Modern top toolbar with clearer hierarchy and primary call-to-action.
  • Zoom controls and smoother node movement on the canvas.
  • Cleaner connector visuals with live drag preview edge.
  • Stronger selected-node contour and refreshed node actions.
  • Airflow Connections quick access from the builder menu.

DAG Generation Behavior (v0)

  • Generates exactly one DAG Python file per pipeline.
  • Uses pipeline-level default args (retry policy, alerts, tags, schedule).
  • Renders module blocks in validated execution order.
  • Chains execution sequentially using task_1 >> task_2 >> task_3 style behavior.
  • Copies generated file to both local generated folder and configured Airflow DAGs path.

Known v0 Scope

  • Sequential orchestration only (no branching or conditional routing yet).
  • One-DAG-per-pipeline workflow.
  • Module logic follows current template patterns and example DAG references.

Next Planned Extensions

  • Parallel branches and conditional execution paths.
  • Reusable sub-pipelines and module presets.
  • Richer validation diagnostics and enhanced preview tooling.
  • Expanded module catalog and deeper runtime observability.
