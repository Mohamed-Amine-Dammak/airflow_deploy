Runtime Monitoring Update

Airflow Orchestration Builder - v0.1

Live DAG execution visibility with task-level status coloring and log viewing directly inside the builder canvas.

Open Builder View v0 Notes

Highlights in v0.1

  • New Run DAG action in the builder header, next to Generate DAG.
  • Run button is enabled only after real readiness checks confirm the DAG is available in Airflow.
  • Added new control-flow components: Parallel Fork, Parallel Join, and Conditional Router.
  • Live polling of DAG run state and task instances after run trigger.
  • Per-node execution contour coloring on the canvas: success (green), failed (red), running (blue), queued (orange), skipped (gray).
  • New Logs button on nodes with task instance data.
  • Bottom logs drawer with task metadata, log content, and manual refresh.

Run & Monitoring Lifecycle

  • Not generated
  • Generating
  • Waiting for Airflow
  • Ready to run
  • Running
  • Run started / Run failed

Header Status Improvements

  • Current run summary badge added in the top toolbar.
  • Run state updates live while Airflow execution progresses.
  • Monitoring stops automatically on terminal states (success, failed, canceled).

Backend API Additions

  • GET /api/runs/<dag_id>/<dag_run_id>/status
  • GET /api/runs/<dag_id>/<dag_run_id>/tasks
  • GET /api/runs/<dag_id>/<dag_run_id>/tasks/<task_id>/logs?try_number=1
  • Airflow auth flow aligned with your script pattern: token + DAG run polling + task list + log fetch.

Task Mapping Reliability

  • DAG generation now returns deterministic node-to-task mapping for runtime monitoring.
  • Supports one-to-many mapping for tag-based Dataform tasks.
  • Mapping is reused by the logs viewer and node status renderer.

Operational Defaults Updated

  • Readiness timeout increased to better match slower Airflow DAG scan intervals.
  • Run polling timeout and interval are now configurable through environment variables.
  • Improved error messages for auth failure, missing runs, missing logs, and API connectivity issues.

Still in Scope for v0.1

  • Builder functionality and saved pipeline format remain backward compatible.
  • Sequential and control-flow behavior from current release remains unchanged.
  • No auth/roles layer yet (planned for future release).
